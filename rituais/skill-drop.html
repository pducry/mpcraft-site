Skill Drop

Show, not tell. O que uma pessoa sabe, todas aprendem. Show, not tell. Lo que una persona sabe, todas aprenden.

Objetivo Objetivo

O Skill Drop parte de uma premissa simples: o conhecimento técnico do time está mal distribuído. Alguém descobriu um atalho no Figma que economiza 20 minutos por dia — e ninguém mais sabe. O Skill Drop resolve isso com o formato mais eficiente de transferência de conhecimento: demonstração ao vivo. El Skill Drop parte de una premisa simple: el conocimiento técnico del equipo está mal distribuido. Alguien descubrió un atajo en Figma que ahorra 20 minutos por día — y nadie más lo sabe. El Skill Drop resuelve esto con el formato más eficiente de transferencia de conocimiento: demostración en vivo.

O sinal de maturidade é quando o time começa a identificar espontaneamente o que vale compartilhar — quando a lógica do "isso seria útil pra todo mundo" vira instinto, não obrigação. La señal de madurez es cuando el equipo empieza a identificar espontáneamente lo que vale compartir — cuando la lógica del "esto sería útil para todos" se vuelve instinto, no obligación.

Setup ao vivoSetup en vivo

0–3 min

Quem apresenta abre as ferramentas, organiza o ambiente e configura o que vai mostrar — ao vivo, sem preparação anterior visível. Faz parte da demonstração. Quien presenta abre las herramientas, organiza el ambiente y configura lo que va a mostrar — en vivo, sin preparación anterior visible. Es parte de la demostración.

Como facilitarCómo facilitar Não use slides. A tela de trabalho real é o palco. O setup ao vivo desmistifica — mostra que é replicável agora, não num ambiente idealizado. No uses slides. La pantalla de trabajo real es el escenario. El setup en vivo desmitifica — muestra que es replicable ahora, no en un ambiente idealizado.

DemonstraçãoDemostración

3–20 min

A técnica em ação. Ferramenta, atalho, fluxo ou método mostrado em uso real. Quem apresenta narra o raciocínio enquanto executa — o porquê de cada passo, não só o como. La técnica en acción. Herramienta, atajo, flujo o método mostrado en uso real. Quien presenta narra el razonamiento mientras ejecuta — el por qué de cada paso, no solo el cómo.

Como facilitarCómo facilitar Incentive perguntas durante — não só no final. Se alguém não entendeu um passo, é o momento certo para repetir devagar. Velocidade de entendimento do time é o ritmo certo. Incentivá preguntas durante — no solo al final. Si alguien no entendió un paso, es el momento correcto para repetirlo despacio. La velocidad de comprensión del equipo es el ritmo correcto.
03

ReplicaçãoReplicación

20–27 min

Todos tentam reproduzir o que viram, com suporte de quem apresentou. O erro é esperado — é parte do aprendizado. Quem trava primeiro revela onde a demonstração precisa de mais clareza. Todos intentan reproducir lo que vieron, con soporte de quien presentó. El error es esperado — es parte del aprendizaje. Quien se tranca primero revela dónde la demostración necesita más claridad.

Como facilitarCómo facilitar Observe onde todo mundo trava. Isso não é falha do time — é dado para quem apresentou melhorar a próxima demonstração. Observá dónde todo el mundo se tranca. Eso no es falla del equipo — es dato para quien presentó para mejorar la próxima demostración.
04

Takeaway

27–30 min

Uma frase por pessoa: "a partir de hoje vou usar isso para...". Específica, não genérica. O takeaway vago não muda nada. O específico muda o fluxo de trabalho de amanhã. Una frase por persona: "a partir de hoy voy a usar esto para...". Específica, no genérica. El takeaway vago no cambia nada. El específico cambia el flujo de trabajo de mañana.

Como facilitarCómo facilitar "Vou usar isso" não é takeaway. "Vou usar auto-layout para os cards de resultado de busca que refaço toda semana" é takeaway. "Voy a usar esto" no es takeaway. "Voy a usar auto-layout para los cards de resultado de búsqueda que rehago cada semana" es takeaway.
